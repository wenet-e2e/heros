# AGENTS.md

## Product Decision

### Desktop-First, Mobile-Supported (Not Web-First)
- We choose desktop clients as the primary form, while also supporting mobile apps.

## Why
- The product is not only for conversation; it must also execute tasks.
- Task execution needs tighter OS-level capabilities (permissions, scheduling, notifications, local integrations).
- A resident desktop form is better suited for "always available" voice interaction and action loops.
- Mobile apps are supported for on-the-go usage and continuity of core voice/task experiences.

## Scope Notes
- This decision is for MVP product shape and interaction quality.
- Platform priority: Desktop first, Mobile alongside, Web not first.
- Web interfaces can still be considered later for lightweight or companion scenarios.
